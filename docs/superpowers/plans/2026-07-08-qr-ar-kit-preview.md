# QR → AR Kit Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the `ar-preview.html` prototype as a production Next.js app where scanning a kit's QR opens `/ar/<kit-id>` and renders the finished 3D-print model in an orbitable viewer that launches AR at true real-world scale.

**Architecture:** Next.js 15 App Router. A static typed registry (`lib/kits.ts`) maps a kit id to model files + metadata. The dynamic route `app/ar/[kit]/page.tsx` (server component) resolves the id and renders a graphite "slicer" shell around a `'use client'` `<Viewer>` that dynamic-imports `@google/model-viewer` on mount. AR routing (Quick Look / Scene Viewer / WebXR) is delegated entirely to `<model-viewer>`. A build-time script generates one QR per kit.

**Tech Stack:** Next.js 15.5, React 19, TypeScript, Tailwind CSS v4, `@google/model-viewer` (pinned), Vitest (pure-logic tests), `qrcode` (QR script). Deploy target Vercel (out of scope for this build).

## Global Constraints

- **Web only, no native.** AR runs through `<model-viewer>` → Quick Look (iOS) / Scene Viewer (Android) / WebXR. Do not add WebAR/marker libraries.
- **Real scale is the point:** `ar-scale="fixed"`, `ar-placement="floor"`. GLBs are assumed authored at true metric scale (1 unit = 1 m).
- **Pin `@google/model-viewer` to an exact version** (no `^`/`~`) in `package.json`. Custom-element + SSR interactions are version-sensitive.
- **`<model-viewer>` is SSR-unsafe** — only ever import/render it inside a `'use client'` component, imported in `useEffect` on mount.
- **Design identity is intentional, not a default template.** Port the prototype's tokens/grid/brackets faithfully. Colors: base `#0E1116`, panels `#141922`/`#161C26`, amber `#FF7A18`→`#FFB04A`, ink `#E9EEF6`/`#8792A6`/`#5A6474`. Fonts: Chakra Petch (display/HUD), Inter (body), JetBrains Mono (all readouts/coords/status).
- **Amber is used sparingly:** AR CTA, status dot, corner brackets, load bar, unknown-kit notice only.
- **Registry field names (do not rename):** `id, name, tagline, glb, usdz, dims:{w,d,h}(mm), printTime, filament`.
- **QR content per kit:** `${BASE_URL}/ar/${id}`.
- **Adding a kit must require only:** 1 registry entry + 2 hosted files + 1 generated QR.
- **Respect `prefers-reduced-motion`:** drop `auto-rotate` and dot/CTA animation.
- **Source of truth for behavior + visuals:** `ar-preview.html` (present in repo root). When in doubt, match it.

`lib/kits.ts`:
```ts
export type Kit = {
  id: string;
  name: string;
  tagline: string;
  glb: string; // Android / WebXR — authored at true metric scale (1 unit = 1 m)
  usdz: string; // iOS Quick Look
  dims: { w: number; d: number; h: number }; // millimetres — drives on-screen callouts
  printTime: string;
  filament: string;
};

// Add one entry per kit. glb = Android/WebXR, usdz = iOS Quick Look.
// dims in millimetres drive the dimension callouts.
// IMPORTANT: author the GLB at true real-world scale so ar-scale="fixed" places it life-size.
export const KITS: Record<string, Kit> = {
  astronaut: {
    id: "astronaut",
    name: "Orbital Astronaut",
    tagline: "1:1 scale — see it stand in your room before you commit the filament.",
    glb: "https://modelviewer.dev/shared-assets/models/Astronaut.glb",
    usdz: "https://modelviewer.dev/shared-assets/models/Astronaut.usdz",
    dims: { w: 470, d: 300, h: 1780 },
    printTime: "18h 40m",
    filament: "312 g",
  },
};

export const DEFAULT_KIT_ID = "astronaut";

export function getKit(id: string): Kit | null {
  if (!id) return null;
  return KITS[id.toLowerCase()] ?? null;
}

export function resolveKit(param: string | undefined): {
  kit: Kit;
  requestedId: string;
  isFallback: boolean;
} {
  const requestedId = (param ?? "").toLowerCase();
  const hit = getKit(requestedId);
  if (hit) return { kit: hit, requestedId, isFallback: false };
  return { kit: KITS[DEFAULT_KIT_ID], requestedId, isFallback: true };
}

export function dimLabel(mm: number): string {
  return `${mm} mm`;
}

export function footprint(kit: Kit): string {
  return `${kit.dims.w} × ${kit.dims.d} mm`;
}
```

- [ ] **Step 1: Create the component**

`components/Viewer.tsx`:
```tsx
"use client";

import { useEffect, useRef, useState } from "react";
import type { Kit } from "@/lib/kits";
import { dimLabel } from "@/lib/kits";
import type { ModelViewerElement } from "@/types/model-viewer";

export default function Viewer({ kit }: { kit: Kit }) {
  const mvRef = useRef<ModelViewerElement | null>(null);
  const [progress, setProgress] = useState(0);
  const [arUnavailable, setArUnavailable] = useState(false);
  const [reducedMotion, setReducedMotion] = useState(false);

  useEffect(() => {
    // model-viewer is SSR-unsafe: register the custom element only on the client.
    import("@google/model-viewer");
  }, []);

  useEffect(() => {
    setReducedMotion(
      window.matchMedia("(prefers-reduced-motion: reduce)").matches
    );
  }, []);

  useEffect(() => {
    const mv = mvRef.current;
    if (!mv) return;
    const onProgress = (e: Event) => {
      const detail = (e as CustomEvent<{ totalProgress: number }>).detail;
      setProgress(detail?.totalProgress ?? 0);
    };
    const onLoad = () => {
      if (!mv.canActivateAR) setArUnavailable(true);
    };
    mv.addEventListener("progress", onProgress);
    mv.addEventListener("load", onLoad);
    return () => {
      mv.removeEventListener("progress", onProgress);
      mv.removeEventListener("load", onLoad);
    };
  }, [kit.id]);

  return (
    <>
      <div
        className="loadbar"
        style={{
          width: `${progress * 100}%`,
          opacity: progress < 1 ? 1 : 0,
        }}
      />
      <div className="stage">
        <div className="frame">
          <span className="br1" />
          <span className="br2" />
          <span className="dim dim-w">
            <span className="cap">W</span>
            <span className="v">{dimLabel(kit.dims.w)}</span>
          </span>
          <span className="dim dim-h">
            <span className="cap">H</span>
            <span className="v">{dimLabel(kit.dims.h)}</span>
          </span>
          <span className="dim dim-d">
            <span className="cap">D</span>
            <span className="v">{dimLabel(kit.dims.d)}</span>
          </span>

          {/* @ts-expect-error custom element typed in types/model-viewer.d.ts */}
          <model-viewer
            ref={mvRef}
            src={kit.glb}
            ios-src={kit.usdz}
            ar
            ar-modes="webxr scene-viewer quick-look"
            ar-scale="fixed"
            ar-placement="floor"
            camera-controls
            touch-action="pan-y"
            {...(reducedMotion ? {} : { "auto-rotate": true })}
            rotation-per-second="16deg"
            interaction-prompt="none"
            shadow-intensity="0.85"
            exposure="1.05"
            environment-image="neutral"
            alt={`3D preview of ${kit.name}`}
          >
            {!arUnavailable && (
              <button slot="ar-button" className="cta" style={{ position: "static" }}>
                <svg
                  viewBox="0 0 24 24"
                  fill="none"
                  stroke="currentColor"
                  strokeWidth={2}
                  strokeLinecap="round"
                  strokeLinejoin="round"
                >
                  <path d="M12 3l8 4.5v9L12 21l-8-4.5v-9L12 3z" />
                  <path d="M12 12l8-4.5M12 12v9M12 12L4 7.5" />
                </svg>
                View in your room
              </button>
            )}
          {/* @ts-expect-error closing custom element */}
          </model-viewer>
        </div>
      </div>
      {arUnavailable && (
        <p className="hint desk" style={{ display: "block" }}>
          ↳ open this page on your phone to place it at real scale
        </p>
      )}
    </>
  );
}
```

> The `@ts-expect-error` on the JSX tag is a pragmatic guard in case the module augmentation is not applied to the `.tsx` scope; if `tsc` reports the directive is *unused*, delete both `@ts-expect-error` comments (the declaration is working and they are not needed).

### Task 8: QR generation script

Batch-generates one QR (PNG + SVG) per kit encoding `${BASE_URL}/ar/${id}`.

**Files:**
- Create: `scripts/qr.ts`
- Create: `public/qr/.gitkeep`
- Modify: `.gitignore` (ensure generated QR PNG/SVG are committed — do NOT ignore `public/qr`)

**Interfaces:**
- Consumes: `KITS` from `@/lib/kits`; `qrcode` package; `BASE_URL` env (default `http://localhost:3000`).
- Produces: `public/qr/<id>.png` and `public/qr/<id>.svg` for every kit; run via `npm run qr`.

- [ ] **Step 1: Create `public/qr/.gitkeep`**

Create an empty file `public/qr/.gitkeep` so the directory exists before the first generation.

- [ ] **Step 2: Create `scripts/qr.ts`**

```ts
import { mkdir, writeFile } from "node:fs/promises";
import path from "node:path";
import QRCode from "qrcode";
import { KITS } from "../lib/kits";

const BASE_URL = process.env.BASE_URL ?? "http://localhost:3000";
const OUT_DIR = path.join(process.cwd(), "public", "qr");

async function main() {
  await mkdir(OUT_DIR, { recursive: true });
  const ids = Object.keys(KITS);
  if (ids.length === 0) {
    console.warn("No kits in registry — nothing to generate.");
    return;
  }
  for (const id of ids) {
    const url = `${BASE_URL}/ar/${id}`;
    const png = path.join(OUT_DIR, `${id}.png`);
    const svg = path.join(OUT_DIR, `${id}.svg`);
    await QRCode.toFile(png, url, { width: 1024, margin: 2 });
    const svgString = await QRCode.toString(url, { type: "svg", margin: 2 });
    await writeFile(svg, svgString, "utf8");
    console.log(`✓ ${id} → ${url}`);
  }
  console.log(`Done. ${ids.length} kit(s) written to public/qr/ (BASE_URL=${BASE_URL})`);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

- [ ] **Step 3: Run the script**

Run: `npm run qr`
Expected: console prints `✓ astronaut → http://localhost:3000/ar/astronaut` and `Done. 1 kit(s) …`; files `public/qr/astronaut.png` and `public/qr/astronaut.svg` exist.

- [ ] **Step 4: Verify the QR resolves**

Open `public/qr/astronaut.png`. Confirm it is a scannable QR (visually a valid code). Optionally decode it to confirm it encodes `http://localhost:3000/ar/astronaut`.
